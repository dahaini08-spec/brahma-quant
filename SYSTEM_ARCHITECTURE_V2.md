## 🔴 串联组件（任一失败=系统停止）

| 组件 | 作用 | 失败影响 |
|------|------|---------|
| `data_cache.py` | 数据入口唯一缓存 | 全系统无数据 |
| `market_state.py` | 体制判断+基础指标 | brahma_brain无法运行 |
| `executor.py` NAV熔断 | 风险红线 | 停止开仓 |
| `executor.py` 持仓上限 | 风险红线 | 停止开仓 |
| `ws_guardian.py` | 持仓止损/止盈 | 持仓无保护 |

## ⚠️ 当前已知缺陷与优化方向

### 缺陷1：并联引擎缺乏降级保护 ❌
**问题**：任何并联引擎抛异常会导致brahma_brain整体崩溃  
**方案**：所有 `extra_data` 引擎调用包裹 `try/except`，异常=返回空默认值  
**优先级**：P0

### 缺陷2：5个孤岛引擎未接入 ⚠️
| 引擎 | 功能 | 接入方案 |
|------|------|---------|
| `execution_quality.py` | 成交质量分析 | 接入executor执行后回调 |
| `error_attribution.py` | 亏损归因 | 接入signal-settler每日运行 |
| `auto_review.py` | 信号自动复盘 | 接入brahma-coordinator-daily |
| `model_health_monitor.py` | ML模型健康 | 接入健康看守-12h |
| `offline_adapters.py` | 离线适配器 | 接入dharma回测链 |

### 缺陷3：watchdog检测机制单点 ⚠️
**问题**：watchdog依赖心跳文件，文件损坏时无法检测  
**方案**：增加 `pgrep` 双重验证（心跳+进程）

### 缺陷4：cron任务无幂等保护 ⚠️
**问题**：gateway重启后部分任务重复触发  
**方案**：给高频任务加运行锁文件 `/tmp/lock_{jobname}`

### 缺陷5：brahma_brain引擎串行调用耗时 ⚠️
**问题**：40+引擎串行调用，每次分析可能超30s  
**方案**：将独立引擎组改为 `concurrent.futures.ThreadPoolExecutor` 并发调用

## 📐 止损设计哲学（梵天铁律）

```
止损 = 结构失效点
止损距离 ≥ ATR(1H) × 1.5
止损 = 最近摆动高/低点 ± 小缓冲

禁止：固定金额止损（如$11/$22）
禁止：止损 < ATR（必被噪音扫掉）
禁止：多档位止损不一致
```
