### 📐 量化工程师视角（统计科学家）

**核心关切：数字的统计质量，结论能否推广**

**样本量充足性分析：**

```python
# 统计功效分析（Power Analysis）
# H0: WR = 50%, H1: WR != 50%, α=0.05, β=0.20（80%功效）
# 最小样本量公式：n = (Z_α/2 + Z_β)² / (2 × effect_size²)

from scipy.stats import norm
import numpy as np

def min_sample(p0=0.5, p1=0.55, alpha=0.05, power=0.80):
    z_a = norm.ppf(1-alpha/2)
    z_b = norm.ppf(power)
    se = np.sqrt(p0*(1-p0))
    return int((z_a*se + z_b*np.sqrt(p1*(1-p1)))**2 / (p1-p0)**2)

# 检测5%差异需要的最小样本
n_5pct = min_sample(0.5, 0.55)  # ≈ 1537
n_10pct = min_sample(0.5, 0.60)  # ≈ 400
n_20pct = min_sample(0.5, 0.70)  # ≈ 103

print(f"检测5%WR差异需要n≥{n_5pct}")   # 1537
print(f"检测10%WR差异需要n≥{n_10pct}")  # 400
print(f"检测20%WR差异需要n≥{n_20pct}")  # 103
```

**各条款样本量充足性裁定：**

| 条款 | 最小周期n | 检测精度 | 充足性 |
|------|---------|---------|--------|
| C1 BEAR禁多 | 67（4H） | ±12% | 🟡 4H边际，15m/1H充足 |
| C2 BULL做多 | 1,695（4H）| ±2.4% | ✅ 三周期充足 |
| C3 BEAR做空 | 23（4H） | ±20% | 🔴 4H严重不足，1H边际 |
| C4 牛市不做空 | 524（4H） | ±4.3% | ✅ 三周期充足 |
| C6 CHOP亏损 | 3,980（4H）| ±0.7% | ✅ 三周期充足 |

**关键统计警告：**
> C3（BEAR做空）的4H样本仅n=23，均值估计误差±20%，任何结论都不可信。1H的n=451勉强可用，但也只能检测>10%的WR差异。这是整个验证矩阵最薄弱的环节。

**多重检验校正：**
本次进行了6个主要假设检验，需要Bonferroni校正：
- 原α=0.05 → 校正后α=0.05/6=0.0083
- 校正后仍显著的条款：C1（三周期）、C4（1H/4H）、C6（三周期）
- C3的1H p值=~1.0，校正后同样不显著

```python
# 问题根因
BTC_4H_ATR_avg = 1800  # 美元（近3年平均）

入场价_example = 100000

合规率 = 2000/2700 = 74%，即：止损比ATR小26%
→ 被市场噪音打出去的概率 = 高

# 后果
每100笔4H信号中：
  约79笔止损过紧，被噪音打出（损失 2%×79 = -158%）
  约21笔止损合理，持仓到信号失效（预期+5%×21 = +105%）
  净预期：-53%（系统性亏损）！

# 修复方案
def calc_sl(entry_price, regime, direction, atr_4h):
    sl_pct = 0.025 if regime in ('CHOP_MID', 'BULL_TREND') else 0.020
    sl_fixed = entry_price * sl_pct
    sl_atr = atr_4h * 1.5
    sl_dist = max(sl_fixed, sl_atr)
    if direction == 'SHORT':
        return entry_price + sl_dist
    else:
        return entry_price - sl_dist
```
